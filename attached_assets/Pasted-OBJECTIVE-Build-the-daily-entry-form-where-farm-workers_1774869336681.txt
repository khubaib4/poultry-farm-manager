OBJECTIVE: Build the daily entry form where farm workers record mortality, egg production, and feed consumption each day

REQUIREMENTS:
- One entry per flock per day (prevent duplicates)
- Auto-show previous day's stock as reference
- Entry for: deaths, eggs by grade, feed consumed
- Toggle between egg count units (individual vs tray of 30)
- Auto-calculate new stock after deaths
- Notes field for observations
- Edit existing entries (same day or past)

VALIDATION RULES:
- Date: required, cannot be future, cannot be before flock arrival date
- Deaths: required, 0 to opening stock
- Eggs (each grade): required, 0 or positive integer
- Feed: required, positive number
- Only one entry per flock per day (unique constraint)

STOCK CALCULATION LOGIC:
- Opening Stock = flock.current_count (which reflects all previous deaths)
- When entry saved: flock.current_count -= deaths
- When entry updated: flock.current_count += old_deaths - new_deaths
- When entry deleted: flock.current_count += deleted_deaths

UI/UX REQUIREMENTS:
- Large, touch-friendly inputs (farm workers may use on tablet)
- Real-time calculations as user types
- Clear visual feedback for warnings
- Quick date navigation (< Previous Day | Today | Next Day >)
- Remember last used egg unit preference
- Auto-focus on first empty field

OUTPUT: Farm workers can record daily entries for each flock. Stock automatically updates based on mortality. Entries can be viewed, edited, and deleted. Previous entries are accessible via history page.
