INVOICE NUMBER FORMAT:
- Pattern: INV-{YEAR}-{SEQUENCE}
- Example: INV-2026-001, INV-2026-002
- Sequence resets each year
- Padded to 3 digits minimum

SALE CALCULATIONS:
```typescript
// Items total
const eggsTotal = (gradeAQty * gradeAPrice) + (gradeBQty * gradeBPrice) + (crackedQty * crackedPrice);
const traysTotal = (gradeATrays * gradeATrayPrice) + (gradeBTrays * gradeBTrayPrice) + (crackedTrays * crackedTrayPrice);
const subtotal = eggsTotal + traysTotal;

// Discount
const discountAmount = discountType === 'percentage' 
  ? subtotal * (discountPercent / 100) 
  : fixedDiscount;

// Total
const total = subtotal - discountAmount;

// Balance
const balanceDue = total - paidAmount;

// Status
const status = paidAmount >= total ? 'paid' : paidAmount > 0 ? 'partial' : 'unpaid';
```

DUE DATE CALCULATION:
- Due Date = Sale Date + Customer Payment Terms Days
- If payment terms = 0, due date = sale date (cash)

VALIDATION RULES:
- Customer required
- At least one item with quantity > 0
- Prices must be positive
- Discount cannot exceed subtotal
- Paid amount cannot exceed total

UI/UX REQUIREMENTS:
- Real-time calculation as user types
- Customer's default prices auto-fill
- Clear visual of what's being charged
- Easy to add items
- Summary always visible
- Prominent "Overdue" indicator

OUTPUT: Sales can be recorded with custom pricing, discounts. Invoice numbers auto-generated. Totals calculated automatically. Sales list shows payment status.
