OBJECTIVE: Allow farm managers to set and manage egg prices by grade, with price history tracking

REQUIREMENTS:
- Set prices for each egg grade (A, B, Cracked)
- Support both per-egg and per-tray pricing
- Track price history with effective dates
- Current prices shown prominently
- Calculate revenue based on daily production × current price

VALIDATION RULES:
- All prices required and must be positive numbers
- Price per tray must equal price per egg × 30 (auto-calculated)
- Effective date cannot be in the future
- At least Grade A price is required

PRICE CALCULATION LOGIC:
- When updating prices, save 3 records (one per grade) with same effective_date
- Current price = record where effective_date <= today, ordered by effective_date DESC, LIMIT 1
- For historical reports, use getPriceOnDate to get correct price

DEFAULT PRICES (for new farms):
- If no prices set, prompt user to set prices
- Suggest typical market rates (can be skipped)

UI/UX REQUIREMENTS:
- Currency formatted with PKR symbol and thousands separator
- Clear distinction between per-egg and per-tray prices
- Show last updated date prominently
- Warn if prices haven't been updated in 30+ days
- Price history sortable by date

OUTPUT: Farm managers can set egg prices for all grades, view price history, and see current prices. Revenue calculations throughout the app will use these prices.
